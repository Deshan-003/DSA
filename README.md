# DSA
a group project in DSA 

Module 1: Smart City Route Planner (Data Structures & Graphs)

Functional Requirements

• Represent locations and roads using a graph (adjacency list or adjacency matrix).

• Allow users to:

    o Add and remove locations
    o Add and remove roads (edges)
  
• Display all connections between locations.

• Use stacks or queues for traversal or route listing operations.

• Store location data using a tree structure (AVL tree / BST / linked structure) before
mapping it into the graph.

• Provide a menu-driven interface with proper input validation.





Module 2: Data Sorter – Sorting Algorithm Comparison Tool

Functional Requirements

• Allow users to:

    o Enter numbers manually, or
    o Generate random datasets
    
• Implement the following sorting algorithms:

    o Bubble Sort
    o Merge Sort
    o Quick Sort
    
• Display:

    o Sorted output
    o Execution time for each algorithm
    
• Display a comparison table summarizing performance results.


Module 3: Algorithm Performance Analyzer (Time Complexity)

Functional Requirements

• Implement and analyze the following algorithms:

    o One searching algorithm (Linear or Binary Search)
    o One sorting algorithm (Bubble / Quick / Merge Sort)
    
• Run algorithms on different input sizes (e.g., 100, 500, 1000 elements).

• Measure execution time using:

    o System.currentTimeMillis() or System.nanoTime()
    
• Display results in a tabular format.
